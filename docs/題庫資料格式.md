# 題庫資料格式

## 檔案位置

- [idiom-bank.js](/Users/al03160310/Documents/成語怪物/idiom-bank.js)

## 難度分區

題庫以三個 key 分層：

- `basic`
- `advanced`
- `master`

每個 key 對應一個陣列，陣列中每筆題目物件至少需要下列欄位。

## 單題欄位（目前）

```js
{
  idiom: "亡羊補牢",
  meaning: "出了問題後立刻補救，仍然來得及",
  hint: "重點是補救，不是放棄",
  funnyMeaning: "先請羊把作業簿叼回來再補洞",
  image: "",              // 可留空字串，未來放圖片網址
  imageAlt: "亡羊補牢 插圖" // 圖片替代文字
}
```

說明：
- `idiom`：題目顯示的成語
- `meaning`：正確答案
- `hint`：題目提示
- `funnyMeaning`：該題專屬搞笑選項
- `image`：題目圖片 URL（目前可先空白）
- `imageAlt`：圖片說明文字

> 即使你只填 `idiom/meaning/hint`，系統也會自動補上預設 `funnyMeaning`、`image` 與 `imageAlt`。

## 搞笑選項規則

- 搞笑選項改為「每題資料驅動」，來源是該題 `funnyMeaning`
- 每題最多放入 1 個搞笑選項（可由設定關閉）
- 若題目沒有填 `funnyMeaning`，系統會依該成語自動生成固定預設值

## 未來接管理介面的建議格式

若未來改成後端或 CMS，建議使用以下 JSON 結構對應：

```json
{
  "basic": [
    {
      "id": "basic-001",
      "idiom": "亡羊補牢",
      "meaning": "出了問題後立刻補救，仍然來得及",
      "hint": "重點是補救，不是放棄",
      "funnyMeaning": "先請羊把作業簿叼回來再補洞",
      "image": "",
      "imageAlt": "亡羊補牢 插圖",
      "tags": ["補救", "常見"]
    }
  ]
}
```

`id`、`tags` 目前不是必填，但很適合之後做搜尋、過濾、排序。
