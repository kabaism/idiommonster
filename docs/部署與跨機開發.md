# 部署與跨機開發

## A. 線上分享給小孩（GitHub Pages）

這個專案是純前端靜態檔，最適合 GitHub Pages。

## 1) 建立 GitHub Repo

```bash
cd /Users/al03160310/Documents/成語怪物
git init
git add .
git commit -m "init idiom monster"
git branch -M main
git remote add origin https://github.com/<你的帳號>/idiom-monster.git
git push -u origin main
```

## 2) 啟用 Pages

1. 打開 GitHub repo
2. `Settings` -> `Pages`
3. `Build and deployment` 選 `Deploy from a branch`
4. Branch 選 `main`，資料夾選 `/(root)`
5. 儲存，等待 1-5 分鐘

完成後通常網址會是：

`https://<你的帳號>.github.io/idiom-monster/`

## 3) 分享方式

直接把網址傳給小孩即可，手機平板都能開。

## B. 在另一台機器繼續用 Codex 開發

核心原則：**用 GitHub repo 當唯一真實來源**。

## 1) 新機器準備

1. 安裝 Git
2. 安裝並登入 Codex（同一個帳號）
3. `git clone` 專案

```bash
git clone https://github.com/<你的帳號>/idiom-monster.git
cd idiom-monster
```

## 2) 日常開發流程

```bash
git pull
# 開發
git add .
git commit -m "描述這次修改"
git push
```

## 3) 建議習慣

- `main` 維持可玩的版本
- 新功能先開分支（例如 `feature/admin-ui`）
- 每次改動都寫清楚 commit 訊息

---

## C. 之後想做「題庫管理介面」可以這樣走

1. 先做本地管理頁（新增/編輯/刪除題目）
2. 管理頁輸出 `idiom-bank.js` 或 JSON
3. 再決定要不要接後端（Firebase/Supabase/自架 API）

先把資料格式固定，比先上後端更穩。
